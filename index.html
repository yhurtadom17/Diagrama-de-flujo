Diagrama de Flujo

Entrada

  • Algoritmo 1. Crear perfil de usuario (nombre, celular, correo electrónico):
  • Verificar formato de los datos
  • Validar que los campos no estén vacíos
  • Guardar temporalmente los datos en caché

  • Algoritmo 2. Seleccionar el origen y destino:
  • Convertir ubicaciones a coordenadas GPS
  • Verificar que el valor del trayecto sea válido

  • Algoritmo 3. Seleccionar tipo de vehículo para movilizarse (bus, etc.):
  • Mostrar opciones de transporte según la ruta
  • Guardar selección del usuario

  • Algoritmo 4. Localizar ubicación actual del usuario:
  • Acceder a la API de geolocalización
  • Mostrar la ubicación en el mapa
  • Solicitar permisos del usuario para acceder a la ubicación

Almacenamiento

  • Algoritmo 1. Guardar datos del perfil (nombre, celular, correo electrónico):
  • Validar y estructurar los datos en formato de base de datos
  • Guardar datos en almacenamiento seguro (base de datos o caché)

  • Algoritmo 2. Guardar preferencias del usuario (tipo de transporte, rutas seguras):
  • Validar y almacenar las preferencias seleccionadas

  • Algoritmo 3. Almacenar datos de rutas descargadas y zonas a evitar (zonas con tráfico, etc.):
  • Descargar y guardar rutas seleccionadas
  • Actualizar zonas a evitar basadas en congestión o peligros

  • Algoritmo 4. Guardar métodos de pago de manera segura:
  • Implementar encriptación para proteger la información
  • Validar los datos de pago y guardarlos en almacenamiento seguro

Procesamiento

  • Algoritmo 1. Procesar ubicación actual y mostrarla en el mapa:
  • Recibir coordenadas GPS
  • Procesar ubicación con API de mapas
  • Mostrar la ubicación en tiempo real

  • Algoritmo 2. Generar rutas según las preferencias del usuario:
  • Tomar las preferencias del usuario (tipo de vehículo, rutas seguras)
  • Calcular la ruta óptima utilizando una API de rutas

  • Algoritmo 3. Detectar congestiones en tiempo real y generar rutas alternativas:
  • Consultar API de tráfico en tiempo real
  • Detectar congestiones y buscar rutas alternativas

  • Algoritmo 4. Ajustar las indicaciones de la ruta conforme el usuario avanza:
  • Monitorear la ubicación en tiempo real
  • Recalcular ruta si se detecta un desvío

  • Algoritmo 5. Distribuir solicitudes concurrentes para optimizar el rendimiento del sistema:
  • Distribuir las solicitudes de cálculo de rutas entre varios servidores
  • Usar técnicas de balanceo de carga para distribuir el procesamiento

Salida

  • Algoritmo 1. Mostrar al usuario las rutas sugeridas con detalles de tiempo y costos:
  • Calcular y mostrar el tiempo estimado de viaje y los costos

  • Algoritmo 2. Permitir al usuario valorar el conductor y guardar su contacto:
  • Ofrecer un sistema de valoración al finalizar el viaje
  • Guardar el contacto del conductor si es necesario

  • Algoritmo 3. Mostrar el informe detallado del viaje (tiempo, costo, etc.):
  • Generar y mostrar un resumen detallado del viaje

  • Algoritmo 4. Mostrar el historial de viajes realizados:
  • Recuperar y mostrar los viajes anteriores del usuario
